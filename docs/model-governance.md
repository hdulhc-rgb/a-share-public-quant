# 模型治理

## 共同生命周期

1. **预注册**：先固定假设、候选、参数预算、主要指标和失败条件。
2. **数据冻结**：记录 as-of、来源身份、可用时间和内容哈希。
3. **验证**：anchored walk-forward 为主，purge/embargo 留出为补充；严格区分信号与执行时点。
4. **约束**：先计算原始目标，再记录跟踪误差、换手和其他约束如何改变目标。
5. **独立校验**：核心收益路径使用第二种实现对账；差异超容差即失败关闭。
6. **证据闭环**：保留全部候选、拒绝原因、回放、归因和 manifest。
7. **人工晋级**：只报告稳定区和证据，不从自动任务直接切换 champion 或产生交易。

## ETF 研究线

固定挑战者包括等权、逆波动、HRP、CVaR 风险预算、最小 CDaR 和收缩均值–风险。核心门禁为多头、总权重 100%、无杠杆、跟踪误差与单边换手上限，以及向量化/逐日事件回测一致性。

## 单一股票期权证据线

期权层只验证时间、报价、流动性、无套利、IV 和模型残差。美式合约要求显式 QuantLib；缺依赖时为 `NOT_EVALUABLE`，不得静默换成欧式模型。期权结果只能生成反事实，不能改写调用方的价格锚点、触发 Covered Call、连接券商或生成订单。
