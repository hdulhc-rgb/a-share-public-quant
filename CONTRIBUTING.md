# Contributing

本仓库接受研究方法、验证、诊断和测试改进，不接受真实组合数据或交易自动化。

## 提交模型变更前

1. 在 Issue 或 PR 中预先写明假设、唯一主要变更、参数预算和失败条件。
2. 说明数据在每个 as-of 时点何时可见，排除幸存者偏差、后验修订和未来函数。
3. 保持现有 champion / baseline，不因一次回测自动晋级挑战者。
4. 用合成 fixture 或公开、可再分发且不含个人信息的数据测试。
5. 运行隐私守卫和两个模块的单元测试。

## PR 最低证据

- 变更目标与非目标；
- 数据契约和时间边界；
- walk-forward / 留出集设计；
- 收益、回撤、CVaR/CDaR、换手、跟踪误差及约束绑定；
- 拒绝候选和 `NOT_EVALUABLE` 路径；
- 双实现一致性或等价的独立校验；
- 对过拟合、依赖缺失和可复现性的说明。

不得提交真实权重文件、持仓、成本、归属计划、税务、账户截图、券商导出、家庭资产负债或私人研报。不得加入券商 API、订单发送或自动晋级逻辑。
